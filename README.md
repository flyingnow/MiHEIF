# HEIC/HEIF 颜色修正批量转换工具说明

## 1. 项目背景

这个工具主要用于处理一种常见问题：

部分 **小米手机拍摄的 HEIC/HEIF 照片**，在导出到其他手机、电脑或部分图像软件后，可能会出现显示异常，例如：

- 在电脑上打开时，整体 **偏绿** 或 **偏红**
- 白色区域显示不正常，颜色发灰、发绿或发粉
- 同一张照片在手机上看正常，但在其他设备上看明显失真
- 导出为 JPG 后显示正常，但原始 HEIC 在某些软件中显示异常

这个问题通常不是照片内容真的坏了，而是 **部分设备或软件对这类 HEIC 文件中的颜色通道解释方式不一致**。简单说，就是图像本身存储的并不是普通意义下的直接 RGB，而在某些读取链路下被“错误当成 RGB 来解释”，最终导致颜色异常。

本工具针对已经确认存在如下映射关系的图片进行修正：

- `Y = G`
- `Cb = B`
- `Cr = R`
- 使用 **full-range YCbCr → RGB** 进行恢复

也就是说，程序会将错误读取出来的三通道重新解释为亮度与色差信息，再还原为正常 RGB，最后重新导出为新的图片文件。

---

## 2. 适用场景

本工具适用于以下情况：

- 你使用 **小米手机** 拍摄了 HEIC/HEIF 照片
- 将照片传到电脑、其他品牌手机或某些软件后，显示颜色异常
- 已经验证这些照片在某些软件中读取时，存在本文档对应的通道映射问题
- 希望批量修正整个文件夹中的照片
- 希望导出为新的 `HEIF/HEIC` 或 `PNG`

本工具尤其适合以下用途：

- 从小米手机导出照片到 iPhone、安卓其他品牌手机
- 在 macOS / Windows 电脑上查看小米 HEIC 照片时偏色
- 想要批量修复照片而不是一张张手工调色

---

## 3. 功能说明

本程序支持：

- 批量扫描一个文件夹中的 `.heic` / `.heif` 文件
- 自动按固定映射关系恢复正确 RGB
- 默认输出为 `HEIF/HEIC`
- 可选输出为 `PNG`
- 支持多进程并行，利用多核 CPU 提升速度
- 支持递归处理子文件夹
- 支持保留原目录结构输出
- 支持跳过已存在文件或覆盖输出

---

## 4. 原理简介

### 4.1 问题本质

正常情况下，一张彩色图片在读取后应当得到标准 RGB：

- `R`：红色通道
- `G`：绿色通道
- `B`：蓝色通道

但某些异常 HEIC 文件在部分软件中读取后，得到的三通道并不能直接视作正常 RGB。

经过测试，这类图片更接近如下解释方式：

- `G` 通道对应亮度 `Y`
- `B` 通道对应蓝色差 `Cb`
- `R` 通道对应红色差 `Cr`

也就是：

```text
Y  = G
Cb = B
Cr = R
```

然后程序使用 **full-range YCbCr → RGB** 公式进行恢复：

```text
R = Y + 1.402 * (Cr - 128)
G = Y - 0.344136 * (Cb - 128) - 0.714136 * (Cr - 128)
B = Y + 1.772 * (Cb - 128)
```

恢复后再重新保存为标准图像文件。

### 4.2 为什么会偏绿或偏红

当一张本应按 YCbCr 解释的图像，被某些软件直接当成 RGB 显示时，就会出现：

- 某个通道明显偏高，导致整体偏绿
- 某个通道明显偏低，导致整体偏红或偏蓝
- 白色区域无法正确显示为中性白

这个工具的作用，就是把这些“读错了的颜色”重新映射回去。

建议项目目录如下：

```text
小米HEIC转换/
├── main.py
├── config.json
├── README.md
├── input/
│   ├── photo1.HEIC
│   ├── photo2.HEIC
│   └── ...
└── output/
```

其中：

- `main.py`：主程序
- `config.json`：配置文件
- `README.md`：使用说明
- `input/`：原始 HEIC/HEIF 图片

## 6. 运行环境

### 6.1 Python 版本

建议使用：

- Python 3.10 及以上

### 6.2 依赖安装

先安装依赖：

```bash
pip install pillow pillow-heif numpy
```

如果你使用的是 Anaconda，也可以在对应环境中安装。

### 6.3 macOS 额外说明

如果你在 macOS 上使用，通常不需要额外操作；但如果 `pillow-heif` 编码 HEIC 遇到问题，可能和本机底层 `libheif` 环境有关。

## 7. 配置文件说明

程序不使用命令行参数，而是通过 `config.json` 读取配置。

一个典型配置如下：

```json
{
  "_comment": "HEIC/HEIF 批量修正配置文件",

  "_input_dir_note": "输入目录：存放原始 HEIC/HEIF 图片的文件夹",
  "input_dir": "/Users/gabriel/Library/Mobile Documents/com~apple~CloudDocs/Code/小米HEIC转换/input",

  "_output_dir_note": "输出目录：保存转换后图片的文件夹",
  "output_dir": "/Users/gabriel/Library/Mobile Documents/com~apple~CloudDocs/Code/小米HEIC转换/output",

  "_output_format_note": "输出格式：可选 heif / heic / png；默认建议 heif",
  "output_format": "heif",

  "_workers_note": "并行进程数；0 或不大于 0 表示自动使用 CPU 核数",
  "workers": 6,

  "_heif_quality_note": "HEIF 质量；-1 表示尽量高质量，95 通常速度和体积更均衡",
  "heif_quality": -1,

  "_png_compress_level_note": "PNG 压缩等级 0~9；数值越大压缩越强，速度越慢",
  "png_compress_level": 9,

  "_recursive_note": "是否递归处理子文件夹",
  "recursive": false,

  "_keep_subdir_structure_note": "是否在输出目录中保持原有子目录结构",
  "keep_subdir_structure": false,

  "_overwrite_note": "是否覆盖已存在的输出文件",
  "overwrite": true
}
```

### 7.1 各字段说明

#### `input_dir`
输入文件夹路径。

这里放原始的小米 HEIC/HEIF 照片。

#### `output_dir`
输出文件夹路径。

修正后的图片会保存到这里。

#### `output_format`
输出格式，可选：

- `heif`
- `heic`
- `png`

默认建议用 `heif` 或 `heic`，因为体积通常更小，更适合照片。

#### `workers`
并行进程数。

- 设为 `0`：自动使用 CPU 核数
- 设为 `4`、`6`、`8` 等：手动指定并行数量

一般不建议设置得过大，否则 HEIC 编码可能造成内存和 CPU 竞争，反而变慢。

#### `heif_quality`
HEIF 输出质量。

- `-1`：尽量高质量
- `95`：通常速度和体积更均衡
- 更低数值：体积更小，但画质下降更明显

#### `png_compress_level`
PNG 压缩等级，范围 `0 ~ 9`。

- `0`：几乎不压缩，文件最大，保存最快
- `9`：压缩最强，文件更小，保存更慢

注意：PNG 本身是无损格式，这里只影响体积和速度，不影响像素质量。

#### `recursive`
是否递归处理子目录。

- `false`：只处理输入目录第一层文件
- `true`：连子文件夹中的 HEIC/HEIF 一起处理

#### `keep_subdir_structure`
是否在输出目录中保留输入目录的子目录结构。

- `false`：所有输出文件直接放到输出目录下
- `true`：尽量保持原有目录层级

#### `overwrite`
是否覆盖已有输出文件。

- `true`：如果输出文件已存在，则覆盖
- `false`：如果输出文件已存在，则跳过

## 8. 使用步骤

### 第一步：准备原始照片

把需要修复的 `.HEIC` 或 `.HEIF` 文件放到 `input/` 文件夹中。

### 第二步：修改配置文件

根据你的目录情况修改 `config.json` 中的：

- `input_dir`
- `output_dir`
- 需要的输出格式
- 并行数

### 第三步：运行程序

在项目目录下执行：

```bash
python main.py
```

### 第四步：查看输出结果

程序处理完成后，修正后的图片会保存在 `output/` 文件夹中。

---

## 9. 输出格式建议

### 9.1 默认推荐：HEIF / HEIC

适合照片批量保存。

优点：

- 文件体积通常比 PNG 小很多
- 更适合手机照片
- 后续继续传回手机也比较方便

缺点：

- 编码速度一般比 PNG 慢
- 依赖本机底层 HEIF 编码环境

### 9.2 可选：PNG

适合以下场景：

- 你想验证修正结果是否正确
- 需要无损保存中间结果
- 想避免再次进行 HEIF 编码

优点：

- 无损
- 通用性强
- 方便在电脑上做二次分析

缺点：

- 对照片来说文件会很大
- 高分辨率照片可能几十 MB 一张
